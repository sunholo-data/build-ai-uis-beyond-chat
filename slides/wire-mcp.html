    UI by reference

    The tool advertises _meta.ui.resourceUri on tools/list; the host fetches the HTML with a separate resources/read. Never embedded in the tool result.

    2

    Two channels back

    ui/message = a synthetic chat turn; ui/update-model-context = state for the next turn. Both are postMessage, not AG-UI — a sandboxed iframe can't ride the agent stream, so the host stashes it in session state. (A2UI, host-rendered, rides AG-UI forwardedProps instead.)

    3

    Sandboxed

    Mounts in a sandboxed iframe on a separate origin (:3457). No host cookies, no host DOM — CSP via HTTP headers, message-passing only.

    Portable UI today, portable context tomorrow. MCP Apps standardized the render — UI by reference, sandboxed, portable across 11 hosts. The interaction → model back-channel hasn't: each host wires it differently.
